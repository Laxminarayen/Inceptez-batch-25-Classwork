4. Adding the Tree (Lambda)

We don't just add the full tree result. We scale it down using a Learning Rate (\(\lambda\)).

New Pred = Old Pred + (\( \lambda \) * Tree Output)

Let's set \(\lambda = 0.1\) (A small step).

Example: ID 1 (Target 50)

Old Pred: 70

Tree Says: -15

\( 70 + (0.1 \times -15) \)

\( 70 - 1.5 = \mathbf{68.5} \)

Did it improve?

Old Error: -20

New Prediction: 68.5

New Error: \( 50 - 68.5 = \mathbf{-18.5} \)

Error Reduced!

Why is it called "Gradient" Boosting?

It sounds fancy, but it's simple calculus logic.

üìâ

Gradient Descent

In math, the "Gradient" points up the steepest slope.

To minimize error, we want to go Down the slope.

The Secret

Mathematically, the Residual (Error) is basically the Negative Gradient.

So when we fit a tree to the residual, we are literally building a map to go "downhill" towards zero error.

Enter XGBoost: "Extreme" Gradient Boosting

GBM works, but it's slow and can overfit. XGBoost is the "Pro Version".

Why "Extreme"?

  1. Extreme Speed: Uses hardware optimization (Cache/Parallelism).
  2. Extreme Pruning (Regularization): It has built-in safeguards to prevent complex trees.

XGBoost Technicality: Regularization

How does XGBoost stop from memorizing noise?

The "Gain" Check

Normally, if a split reduces error by 0.0001, GBM takes it.

XGBoost has a parameter called Gamma (\(\gamma\)).

If (Improvement - \(\gamma\)) < 0
THEN: Don't Split!

Gamma acts like a "Minimum Ambition" threshold.

If the improvement isn't big enough, XGBoost Prunes (cuts off) the branch.

Decoding the Names

Term Meaning
Gradient We use the "slope" of the error function (Residuals) to guide us.
Boosting Combining weak models (Trees) sequentially to make a strong one.
Extreme Added hardware speed + aggressive regularization (Pruning) to push limits.

Final Review: The Process

  1. Start: Guess the Mean.
  2. Calc: Find Residuals (Target - Guess).
  3. Build: Create a Tree to predict those Residuals.
  4. Update: \( \text{New Guess} = \text{Old Guess} + (\lambda \times \text{Tree}) \).
  5. Repeat: Find new residuals from the new guess.

XGBoost does this, but checks "Is this tree worth it?" (Gamma) before adding it.
